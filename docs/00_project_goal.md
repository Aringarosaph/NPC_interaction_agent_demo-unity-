# 00 项目目标与范围冻结

## 冻结目标

做一个求职作品集级别的小型 NPC RAG Agent demo，而不是生产级剧情系统。

玩家侧效果：

1. Unity 白盒场景中有主角和三个 NPC 白模盒子/胶囊体。
2. 玩家靠近任一 NPC 到 3 米内，视为正在与该 NPC 对话。
3. 玩家在输入框打字并发送。
4. 主角头上显示玩家输入气泡。
5. 后端根据 `npc_id` 检索对应 NPC 的人设、知识、记忆。
6. NPC 通过 1-3 个短气泡回应。
7. 回答受该 NPC 知识库限制，不能串到其他作品世界。

## 明确不做

- 不做大型知识库。
- 不做在线多用户服务器。
- 不做复杂 NPC 自动行为树。
- 不做美术资产、角色模型、语音、口型。
- 不做商业级版权角色还原。
- 不做复杂工具调用 Agent。

## 核心展示点

- 数据驱动 NPC 人设。
- 小型 RAG 检索。
- NPC 知识边界控制。
- JSON 结构化短句输出。
- Unity 对话气泡表现。
- 简单长期记忆。
- 可测试、可录屏、可讲解。
